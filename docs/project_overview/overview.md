# 项目总览

## 1. 项目在做什么（问题定义）
本项目当前聚焦“基于 Isaac Sim 的 FR3 + 原装夹爪方块抓放闭环”。核心问题是：如何在较低人工成本下，先跑通一条可执行、可复现、可验证的具身操作最小链路，再以该链路为基线扩展到更复杂末端和任务。当前阶段先完成 FR3 抓取单方块并放到指定位置，形成仿真到真机最小验证路径。

## 2. 系统边界（负责什么 / 不负责什么）
### 负责
- 在 Isaac Sim 中构建 FR3 + 原装夹爪 + 方块 + 目标放置区的可运行场景。
- 实现脚本化抓放原语并可重复执行最小任务回合。
- 建立围绕抓放任务的数据采集、最小训练和仿真评估流程。
- 完成一次 FR3 真机最小抓放验证并输出执行记录。

### 不负责
- 当前阶段不实现智元灵巧手控制与能力建模。
- 当前阶段不覆盖复杂自动化测试点位寻找与多任务泛化。
- 不提供通用机器人硬件设计与制造能力。

## 3. 输入 / 输出（来源、格式、去向）
### 输入
- 场景与机器人模型：FR3 机器人模型（URDF/USD）与方块模型资源。
- 任务与控制配置：方块初始位、目标放置位、控制参数、随机化参数（YAML/JSON/脚本参数）。
- 真机接入配置：通信参数、安全限幅、执行阈值（ROS 或 SDK 配置）。

### 输出
- 抓放回合样本：状态、动作、时间戳等结构化序列数据。
- 训练与评估产物：最小策略模型、仿真评估结果、成功率统计。
- 真机验证记录：执行日志、误差说明、失败案例与改进建议。

## 4. 目标场景与业务价值
- 当前目标场景：FR3 使用原装夹爪完成单方块抓取并放到指定位置。
- 未来扩展方向：切换智元灵巧手、自动化测试点位寻找、复杂操作任务。
- 业务价值：
  - 快速形成可验证的最小闭环，降低“方案讨论无法落地”的风险。
  - 为后续灵巧手和复杂任务扩展提供稳定基线。
  - 为团队协作提供可追溯的步骤化推进样板。

## 5. 关键实现方式（主流程、核心依赖、运行方式）
### 主流程
1. Step 1 重写并对齐文档基线，明确步骤与门禁。
2. Step 2 完成仿真场景搭建（FR3、方块、目标区与参数基线）。
3. Step 3 跑通脚本化抓放原语，验证最小动作链路。
4. Step 4 建立抓放数据采集流水线并稳定落盘。
5. Step 5 完成最小训练与仿真评估。
6. Step 6 完成 FR3 真机最小验证并产出误差分析。

### 核心依赖
- 仿真平台：Isaac Sim。
- 机器人平台：Franka FR3 + 原装夹爪。
- 算法基线：最小强化学习训练流程（PPO 可选）。
- 通信方式：ROS 或厂家 SDK（按真机环境适配）。

### 运行方式（当前规划）
- 计划侧：`docs/project_plan/` 负责步骤推进、门禁与验收。
- 总览侧：`docs/project_overview/` 负责项目状态、结构导航与统一变更说明。
- 执行侧：严格单步串行推进，未通过当前步骤验收不得进入下一步。

## 7. 风险与限制（假设、限制、失败模式、降级方案）
### 前提假设
- 具备可运行 Isaac Sim 的环境与 FR3 基础模型资源。
- 真机 FR3 具备最小接入条件与安全保护机制。

### 已知限制
- 仿真与真实环境存在不可消除差异（接触、摩擦、时延）。
- 当前阶段以“最小闭环可跑通”为目标，不追求一次性最优策略。
- 末端仍为原装夹爪，灵巧手能力不在当前实现范围。

### 失败模式
- 抓放原语在仿真中不稳定，导致数据质量不足。
- 真机桥接配置不完整，导致动作无法安全下发。
- 步骤文档与执行状态不同步，影响推进判定。

### 降级方案
- 仿真抓放不稳定时，先回退到脚本化单回合可复现再扩采样。
- 真机接入受阻时，标记当前步骤阻塞并保留仿真闭环证据链。
- 文档不一致时，以 `master-plan.md` 当前步骤为真源统一回写。
